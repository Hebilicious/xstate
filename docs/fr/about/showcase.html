# Vitrine

De nombreux développeurs et équipes utilisent XState dans la nature (opens new window), pour aider à contrôler divers aspects de leurs applications, dans le frontend et le backend, et dans de nombreux domaines différents. frameworks (ou même sans framework !) :

  • Le project Hashicorp Vault project (opens new window) uutilise XState pour contrôler les flux d'utilisateurs contextuels afin de permettre aux nouveaux utilisateurs d'être connectés instanément. Les machines sont dans le référentiel open-source (opens new window).
  • Le jeu Service Workies (opens new window) a de nombreux niveaux dont les flux sont contrôlés par XState. - https://twitter.com/geddski/status/1090404900318502912
  • "Chez bridge21.com (opens new window), nous réécrivons notre site en utilisant XState 3 et react-automata. C'est une révolution.
  • KongHQ (opens new window) l'a "... déployé en production à Kong pour gérer les états de formulaire (erreur, chargement, modal, récupération des machines, etc.) Notre application Vue (opens new window)"
  • "Utilisation intensive de XState 4 dans l'application frontale de 4D Healthware (opens new window).
  • "Nous utilisons XState chez Meru Health (opens new window) dans notre application React Native pour gérer de nombreuses interactions.
    • Il y a aussi une présentation pertinente de l'utilisation de XState à React Helsinki (opens new window)
  • "J'ai utilisé XState dans une application tactile pour le Royal Tyrrell Museum (Alberta - Canada) et j'ai documenté son utilisation dans cet article : ["Modeling a Screensaver with a Statechart, a real use case"](https://medium.com/ @carloslfu/modeling-a-screensaver-with-a-statechart-a-real-use-case-f57301682570)."
  • "Utilisé XState pour la navigation d'une application Web d'analyse de fûts (bière) [BinaryBeer] (https://binarybeer.io/)."
